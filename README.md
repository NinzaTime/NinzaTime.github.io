## 🖥️ System Architecture

A technical writing showcase built with a focus on **Glassmorphism**, **Persistence**, and **Low-Level Aesthetics**. This portfolio leverages modern CSS-in-JS logic to maintain state across a multi-page static environment.

### Core Components
* **`index.html`** | Root directory and entry point.
* **`reflection.html`** | Academic synthesis and course reflection.
* **`project.html`** | Project 2: Global Revision (Major Artifact).
* **`minor.html`** | Segmented minor writing revisions.
* **`style.css`** | The "Amethyst" design system (Neon-Purple/Dark Mode).
* **`script.js`** | High-speed decode logic and uptime persistence.

## ⚡ Technical Specifications

### 1. Persistence & State Management
Utilizes `localStorage` to bridge the gap between static pages:
* **Theme Continuity:** Seamless dark/light mode transitions that persist on reload.
* **Persistent Uptime Engine:** A real-time session counter tracking years, months, days, and seconds since the initial system boot.

### 2. Visual Interaction Logic
* **Fast-Decode Reveal:** A high-speed character scrambling algorithm for titles, optimized for a tech-heavy UI feel.
* **Dynamic Particle System:** Interactive mouse-trail sparkles with automatic DOM cleanup to ensure performance stability.
* **Shimmer Gradients:** CSS-animated linear gradients for headers, providing a "living" UI atmosphere.

### 3. Layout & UX
* **Bento-Grid Architecture:** Content structured within glassmorphic bento cards, featuring optimized shadows and static-hover states for readability.
* **Pedagogical Highlighting:** Custom `.revision-mark` system with mode-specific alpha transparency to showcase global and minor revisions clearly.
* **Hardware Acceleration:** Navbar and UI elements use `translateZ(0)` and `will-change` to prevent scrolling jitter.
